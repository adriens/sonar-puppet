Nested case statements are difficult to understand because you can easily confuse the case matchers of an inner case statement as belonging to an outer statement. Therefore nested case statements should be avoided. The same also applies for nested selector statements or selector statements nested in case statements.

Non compliant Code Example

case $var1 {
'abc': {
  include pup
}
'def', 'ghi': {
  case $var2 {
'jkl': {           # Bad indentation makes this ever harder to understand
  include pup
}
'mno': {
  include pup2
}
}
}
default: {
  include pup3
}
}
