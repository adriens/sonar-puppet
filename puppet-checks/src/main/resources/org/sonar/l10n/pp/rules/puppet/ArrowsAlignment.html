For better code readability, all the arrows (hash rocket: => and plusignment: +>) should be aligned in:

  • Resourceâ€™s attribute/value list
  • Hash's key/value list
  • Selector's case/value list
Also, arrows should be placed one space ahead of the longest attribute/key/case of the list.

Noncompliant Code Example

exec { 'cmd':
  path => '/usr/bin', cwd => '/tmp',   # Noncompliant: there should be only one attribute/value pair per line
}

exec { 'cmd':
  path => '/usr/bin',
  cwd     => '/tmp',      # Noncompliant, some whitespaces should be removed before the hash rocket
}

exec { 'cmd':
  path   => '/usr/bin',   # Noncompliant, the hash rocket should be placed one whitespace ahead of 'path'
  cwd    => '/tmp',       # Noncompliant, the hash rocket should be placed one whitespace ahead of 'path'
}

myresource { 'test':
  ensure => present,
  myhash => {
   'key1' => 'value1',
   'key2'   => 'value2',  # Noncompliant, some whitespaces should be removed before the hash rocket
  },
}

Compliant Solution

exec { 'cmd':
  path => '/usr/bin',
  cwd  => '/tmp',
}

myresource { 'test':
  ensure => present,
  myhash => {
   'key1' => 'value1',
   'key2' => 'value2',
  },
}
