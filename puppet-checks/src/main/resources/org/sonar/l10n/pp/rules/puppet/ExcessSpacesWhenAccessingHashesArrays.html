Since Puppet 4, the space between a variable and a left bracket is significant. Indeed, Puppet outputs different results if there is a space:

  • $var [3] outputs first the value of $var, then a literal array with the single value 3 in it.
  • $var[3] outputs index 3 of the array referenced by $var.
As 99% of the time, you mean the second bullet point, remove the whitespaces to be compliant with Puppet 4 new syntax.

Noncompliant Code Example

$var [3]

Compliant Solution

$var[3]
