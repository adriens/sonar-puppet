Resource default statement syntax is resource type, capitalized. If the resource type has a namespace separator (::) in its name, every segment must be capitalized. E.g., Concat::Fragment.

Rule ClassAndDefineNamingConvention enforcing the following naming convention: ^([a-z][a-z0-9_]*::)*[a-z][a-z0-9_]*$ , resource default statements should comply with the following naming convention: ^([A-Z][a-z0-9_]*::)*[A-Z][a-z0-9_]*$ . If not, the catalog compilation does not fail and the resource default statement is useless because it does not match any resource type.

Noncompliant Code Example

Foo::BAR {
  ensure => present,
}

Compliant Solution

Foo::Bar {
  ensure => present,
}
