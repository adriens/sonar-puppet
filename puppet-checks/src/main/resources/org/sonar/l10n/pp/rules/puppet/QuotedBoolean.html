Boolean values (true and false) behave differently when quoted ('true' and 'false'), which can lead to a fair bit of confusion. Quoting booleans becomes even more error prone with Puppet 4 as comparing booleans with quoted string always returns false. Thus, you should never quote booleans.

Noncompliant Code Example

file { '/tmp/foo':
  purge => 'true',
}

if $is_virtual == 'false' { }

Compliant Solution

file { '/tmp/foo':
  purge => true,
}

if $is_virtual == false { }
